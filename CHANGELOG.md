# Changelog

Все заметные изменения по версиям.

## [0.3.0] — 2025-03-10

### Добавлено

- **verify** — команда проверки целостности версии (`pyembed verify [version]`): наличие python.exe и ключевой DLL.
- **info** — информация о версии: путь, наличие pip, размер каталога, в PATH или нет (`pyembed info [version]`).
- **cache** — управление кэшем архивов: `pyembed cache list`, `pyembed cache clear [version]`.
- **upgrade-pip** — обновление pip в установленной версии (`pyembed upgrade-pip [version]`).
- **which** — путь к python.exe для версии (`pyembed which [version]`), удобно для скриптов и CI.
- **Кэш архивов** — скачанные zip хранятся в `pythons/.cache/` и переиспользуются при повторной установке той же версии/архитектуры.
- **bootstrap.bat / bootstrap.ps1** — на машине без Python скрипт сам скачивает embed 3.12.0 и запускает менеджер.
- **CHANGELOG.md** — этот файл.
- Конфигурация **pyright** (режим strict) и **ruff** для типов и стиля.
- Сборка одного **exe** через PyInstaller (см. `scripts/build_exe.py` или инструкцию в README).
- **GitHub Actions** — прогон тестов на Windows.
- Подсказки в меню: при вводе версии показываются недавно использованные версии.
- Тесты для **path_env** (add/remove/list PATH с моком реестра).
- В README: раздел **FAQ** (нет сети, ARM64, кэш, удаление менеджера, кодировка) и **примеры для CI** (GitHub Actions, GitLab CI).
- В интерактивном меню: **8** — инфо о версии, **9** — кэш (список/очистка), **10** — обновить pip, **11** — выход.
- Тесты для CLI (cache list, info, upgrade-pip при отсутствующей версии) и для кэша (get_cache_dir, list_cache, clear_cache).
- **install -y/--yes** — не выводить подсказку «Добавить в PATH» после установки (для скриптов и CI).
- Ограничение размера страницы при **list -a** (макс. 2 МБ), чтобы не раздувать память при сбое сервера.
- **Повтор загрузки** при сбое (до 3 попыток для архива и get-pip).
- Опциональная поддержка **requests**: при установленном пакете `requests` он используется для загрузки списка версий (`list -a`).
- При 404 (версия не найдена) подсказка **«Возможно, вы имели в виду: 3.14.3, …»** из списка доступных версий.
- После успешного **install** — ссылка **«Что нового»** (docs.python.org/whatsnew).
- При **uninstall** перед подтверждением показывается **размер** («Освободится ~X МБ»).
- Флаг **-c/--copy** у команд **which**, **path show**, **use** — копирование пути в буфер обмена (Windows).
- Команда **doctor** — проверка: доступность python.org, место на диске, целостность установленных версий, дубликаты в PATH (Windows).
- В embeddable при отсутствии модуля **venv** — автоматический fallback на **virtualenv** через pip (п. 6 меню и команда venv).
- **scripts/backup_sources.py** — бэкап исходников проекта (без pythons, кэша, venv) в zip `backups/pyembed-vYY.MM.DD.HHMM.zip`; функция **run_backup()** для вызова из кода.
- **scripts/backup_gui.py** — GUI для бэкапа (tkinter): кнопка «Создать бэкап», открыть папку бэкапов; запуск: `python scripts/backup_gui.py`.
- **scripts/pyembed_gui.py** — GUI для всего менеджера (tkinter): список установленных версий (★ по умолчанию, [PATH]), кнопки «Установить», «Удалить», «По умолчанию», «В PATH»/«Убрать из PATH», pip/add-pip/upgrade-pip, venv, пакеты, кэш, Doctor, IDE (путь), «Доступные версии»; вывод команд в окне; запуск: `python scripts/pyembed_gui.py`.
- **path fix-duplicates** — команда удаления дубликатов записей в PATH пользователя (Windows): `pyembed path fix-duplicates`; порядок записей сохраняется (остаётся первое вхождение). В **doctor** при обнаружении дубликатов выводится подсказка «Исправить: pyembed path fix-duplicates». В GUI — кнопка «Убрать дубликаты» в группе PATH.
- **doctor --fix** — флаг автоисправления: при дубликатах в PATH выполняет `path fix-duplicates` и убирает проблему из отчёта.
- В интерактивном меню пункт **7 (PATH)**: подменю «1 — Добавить/убрать версию» и «2 — Убрать дубликаты в PATH».
- **copy** — команда копирования установленной версии в папку (например `C:/Python/3.15`) с добавлением в PATH: `pyembed copy <версия> [папка] [--force] [--no-path]`; по умолчанию папка `C:/Python/<версия>`. В меню — пункт **11** «Копировать версию в папку и в PATH»; в GUI — кнопка «Копировать в папку».
- **Прогресс при copy** — перед копированием выводится «Копирование X в Y...».
- **doctor: несуществующие каталоги в PATH** — проверка записей PATH на существование на диске; при наличии вывод списка (до 5 + «и ещё N»); **doctor --fix** удаляет такие записи из PATH.
- **--dry-run** для **install** и **copy** — вывод плана без загрузки/распаковки или копирования (`pyembed install 3.15 --dry-run`, `pyembed copy 3.15 C:/Python/3.15 --dry-run`).

### Изменено

- Установка (install) использует кэш: при наличии архива в `.cache` загрузка пропускается.
- Перед распаковкой проверяется целостность zip (битый архив удаляется из кэша, при следующем запуске — повторная загрузка).
- Перед загрузкой проверяется свободное место на диске; при прерывании (Ctrl+C) частично скачанные файлы и каталоги удаляются.

- Тесты (pytest): version_util, local, download, config.
- pyproject.toml, entry point `pyembed`.
